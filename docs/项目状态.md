## 6. 风险与应对

| 风险 | 触发条件 | 应对 |
|---|---|---|---|
| Nikon 相机 Wi-Fi 热点连接不稳定 | Android 端 `Socket.connect` 超时或闪断 | 先做 loopback 集成测试，再逐步加重试与超时策略 |
| 不同 Nikon 机身 opcode 差异 (Z 系列 vs D 系列) | `getObjectHandles` 返回空 / `getObjectsMetaData` 不支持 | `_classifyObjectFormat` 回退逻辑 + 记录不兼容机型到兼容性列表 |
| Windows 无法搭建 iOS 构建环境 | iOS 端验证必须等 macOS | Phase 2 先专注 Android；iOS 端留到 Phase 3 |
| PTP/IP 数据集解析 (parseDeviceInfo / parseObjectInfo) 占位未完整实现 | 真机连上后 `deviceInfo` / `objectInfo` 字段为空 | Phase 2 按真机日志补全 dataset 解析 |
| Android scoped storage 限制 RAW 入相册 | Phase 3 | 评估 `gal` 包；不支持则降级到 "写入 app 专属目录 + 提示用户" |

## 8. 决策日志 (Decision Log)

| 日期 | 决策 | 备选 | 选择原因 |
|---|---|---|---|
| 2026-07-21 | 走 Flutter 不走 KMP | KMP / 纯 Native Kotlin | 用户希望双端同步出包；iOS 体验可降级 (有原 iOS 项目作参考) |
| 2026-07-21 | 用 Riverpod 2.x | Bloc / Provider | 用户在前一轮已点名 Riverpod |
| 2026-07-21 | 用 freezed 写 Domain | 手写 data class | 替代 `==`/`hashCode`/`copyWith`/`fromJson` 样板 |
| 2026-07-21 | 首版只支持 Nikon | 多品牌并行 | Phase 5 留扩展点；首版避免范围蔓延 |
| 2026-07-21 | 不实现 Live Activity | 写 platform channel 模拟 | Android 无对应；通知 + foreground service 是务实降级 |
| 2026-07-21 | 隐私邮箱 (noreply) | 真实邮箱 (qq.com) | 防止 commit 历史泄露真实邮箱 |
| 2026-07-21 | `pub.dev` 走国内镜像 (`pub.flutter-io.cn`) | 直连 pub.dev | `pub.dev` 域名解析失败，必须用镜像 |
| 2026-07-21 | Phase 0 只创建 Android 平台 | iOS + Android | Windows 上没法编 iOS，先跳过；iOS 部分靠 GitHub Actions 云编或将来借 Mac |
| 2026-07-21 | 用 `dart analyze` 而非 `flutter analyze` | `flutter analyze` | `flutter analyze` 在中文路径下有 LSP 通信 bug；`dart analyze` 工作正常 |
| 2026-07-21 | Dart 包名用 `viewfinder` 而非 `Viewfinder` | 保持目录名 | Dart 包名强制 snake_case lowercase；目录名可保留 `Viewfinder` 大写 |
| 2026-07-21 | CameraCapability 拆出独立文件 | 同文件 | freezed 2.5.8 跟 Dart 3.12 不兼容，混合 enum + freezed class 同文件会触发解析错误 |
| 2026-07-21 | enum 内 getter 改成顶级函数 | enum 内 getter | 同上原因；Phase 1 后期可等 freezed 升级后改回 |
| 2026-07-22 | Phase 1 协议层用单 PtpipSession 类（合并 iOS 三 extension） | 拆成 3 个 class | 避免状态共享循环依赖 |
| 2026-07-22 | 用 `PTPIPCodec` 类名而非 `PTPIPBinary` | 保留 `PTPIPBinary` | 与 iOS 命名区分，代码生成器合并需求 |
| 2026-07-22 | `_collectDataStream` 传入 `expectedTxId` 校验 | 纯内部递增 | 对齐 iOS 的 transactionID 防守式校验 |
| 2026-07-22 | `_requestResponseOnly` 校验 txId 先于 response code | code 先于 txId | 与 `_collectDataStream` 校验顺序一致 |
| 2026-07-22 | FakePtpipSocket 用 `scriptedErrors` 注入错误场景 | 继承 + override | 比子类化更灵活，单文件含全部错误注入 |
