# prd.md — 产品需求文档

> **Viewfinder (取景器)**：通过 Wi-Fi 热点连接相机 (首期 Nikon)，浏览照片列表并批量下载到手机的 Flutter 跨端 app。
>
> 本文档只回答"做什么、做到什么程度算完成"。**怎么做**详见 [`arc.md`](arc.md) 和 [`Viewfinder方案.md`](Viewfinder方案.md)。

---

## 1. 一句话定位

**摄影师连相机 Wi-Fi 后，能在手机上快速浏览相机里的照片，并批量下载到相册。**

---

## 2. 痛点 / 用户 / 场景

### 2.1 痛点

- 相机原厂 app (Nikon SnapBridge 等) **连接慢、断线频繁、批量下载体验差**
- 摄影师外出拍摄后想快速把 RAW / JPEG 取走，但不愿打开电脑
- 现存第三方工具多为桌面端，手机端空白

### 2.2 用户

- **核心用户**：使用 Nikon (后续 Sony / Canon / Fujifilm) 相机的摄影师 / 摄影爱好者
- **使用环境**：野外 / 影楼 / 旅拍现场，**无 Wi-Fi 网络基础设施**，必须依赖相机自带热点
- **技术水平**：会用手机、知道相机 Wi-Fi 怎么开；**不一定懂 IP / 端口 / 网络协议**

### 2.3 典型场景

| 场景 | 用户动作 | 期望结果 |
|---|---|---|
| S1 | 拍完一天，回家连相机热点，开 app | 1 分钟内看到相机里今天拍的 200 张照片缩略图 |
| S2 | 翻看缩略图，挑出想精修的 30 张 | 点击多选 → 一键批量下载 → 通知中心显示进度 |
| S3 | 下载进行中手机息屏 | 锁屏 / 通知中心持续显示下载进度，不打断 |
| S4 | 下载途中相机突然断电 / 走远断线 | app 提示断线，**已下载的不丢失** |
| S5 | 想把 JPEG 先快速预览，RAW 后续慢慢下 | 设置里勾"JPEG 优先"，JPEG 先下载完成 |
| S6 | 想确认下载的照片确实到了相册 | 跳到系统相册看到新照片 |

## 3. 核心功能 + 验收标准

每条验收标准都是 **DOR (Definition of Ready)**：写到这条时，agent 就能知道"做什么算完"。

### F1 — 连接相机

| 项 | 验收 |
|---|---|
| 默认端点 | `192.168.1.1:15740` (CIPA PTP/IP 标准)，用户在设置页可改 |
| 连接过程 | 进度反馈：搜索 → 握手 → 打开 session → 获取设备信息 |
| 失败提示 | 明确分类：网络不通 / 端口拒绝 / 协议握手失败 / 设备忙 |
| 重连 | 自动重试 3 次；用户可手动重试 |

### F2 — 浏览照片列表

| 项 | 验收 |
|---|---|
| 列表加载 | 相机端 GetObjectHandles 全量拉取 (首期不分页) |
| 缩略图 | 显示相机缩略图 (JPEG)；本地有缓存则优先用本地 |
| 筛选 | 按类型：RAW / JPEG / MOV 复选 |
| 排序 | 按拍摄时间倒序 (最新在前) |
| 性能 | 200 张列表 < 3 秒展示；缩略图边滚动边加载，无明显卡顿 |

### F3 — 批量下载

| 项 | 验收 |
|---|---|
| 多选 | 长按进入选择模式，点击勾选；底部显示已选数量与总大小 |
| 队列 | 选中的所有照片加入下载队列，**按 JPEG 优先 / 拍摄时间分组保持原顺序** |
| 后台 | app 进入后台时，下载不中断 (iOS background fetch / Android foreground service) |
| 进度 | 通知中心 (Android) 实时显示百分比 + 已传 / 总计；iOS 仅静态文字通知（**不支持进度条**），用户需打开 app 查看进度 |
| 取消 | 用户可取消单张或整个队列 |
| 断线恢复 | Wi-Fi 丢失后已下载的入库完成；未下载的排队等待重连 |

### F4 — 写入相册

| 项 | 验收 |
|---|---|
| 平台 | iOS PhotoKit / Android MediaStore |
| 格式 | JPEG 直接入库；RAW 写入 `Pictures/Viewfinder/` 目录；MOV 同 RAW |
| 元数据 | 保留 EXIF (拍摄时间、相机型号、镜头等) |
| 通知 | 入库完成发系统通知 |

### F5 — 设置

| 项 | 验收 |
|---|---|
| 连接 | 改 host / port / transport mode |
| 下载 | JPEG 优先开关、自动入库开关 |
| 关于 | 版本号、开源许可、协议致谢 |

## 4. 非功能需求

### 4.1 性能

| 指标 | 目标 |
|---|---|
| app 冷启动到首屏 | < 2 秒 (iPhone 12 / Pixel 6 基线) |
| 列表加载 200 张 | < 3 秒 |
| 单张 JPEG 下载 (10MB) | < 5 秒 (相机热点实测) |
| 包大小 | APK < 30 MB；IPA < 40 MB |

### 4.2 兼容性

| 平台 | 最低版本 |
|---|---|
| iOS | 16.0 |
| Android | API 24 (Android 7.0) |

### 4.3 安全 / 隐私

| 项 | 策略 |
|---|---|
| 网络 | 仅本地局域网，**不连任何外部服务器** |
| 数据 | 不上传任何照片 / 元数据到第三方 |
| 权限 | 相册写入、网络访问、通知 (运行时申请) |
| 凭证 | 无登录 / 无 token / 无 API key |

### 4.4 可维护性

- Dart 代码遵守 `flutter_lints`
- 关键逻辑 (协议层) **单测覆盖率 ≥ 80%**
- 每个 Phase 结束产出可演示的功能切片

## 5. 视觉与页面框架

沿用原 iOS 项目 `NikonConnectIOS` 的 **Claude-style 极简美学**：

- **背景色**：暖白 `#F9F9F8`
- **强调色**：琥珀金 (Amber Gold)
- **按钮**：黑色圆角胶囊 (Capsule)
- **字体**：系统默认 + 中文苹方/思源黑体

### 5.1 页面清单

| 页面 | 路由 | 主要组件 |
|---|---|---|
| 连接页 | `/` | `LensGlowView` (镜头呼吸光晕)、品牌滚轮标题 |
| 相册浏览 | `/browser` | 网格 + 缩略图 (`ShimmerView` 占位)、多选模式 |
| 下载管理 | `/downloads` | 队列列表、进度条、删除/重试按钮 |
| 设置 | `/settings` | 列表 + Switch / 输入框 |

### 5.2 触觉反馈

- 主按钮按下：Medium
- 勾选照片：Light
- 连接成功：Success
- 连接失败 / 下载出错：Error

## 6. 边界 / 不做 (Out of Scope)

明确不做，避免范围蔓延：

| 不做 | 原因 |
|---|---|
| iOS Live Activity | Android 无对应；跨端统一为通知 + foreground service |
| macOS / Windows / Web | 跨平台 UI 投入产出比低 |
| 编辑照片 | 不是传输工具的职责 |
| 云同步 | 隐私考量 + 复杂度 |
| Sony / Canon / Fujifilm | 留作 Phase 5 占位，**首版只实现 Nikon** |
| 实时取景 (Live View) | 需要视频流协议，复杂度高，后续评估 |
