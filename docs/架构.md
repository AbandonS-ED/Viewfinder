## 1. 技术栈决策

完整对比与理由见 `Viewfinder方案.md` §2。此处只锁死结论（**Phase 2 已落地部分加粗**）：

| 层 | 选型 | 备注 | 状态 |
|---|---|---|---|
| 语言 | Dart 3.x | Flutter 默认 | ✅ |
| Framework | Flutter stable | 用户选 Flutter (vs KMP) | ✅ |
| 状态管理 | **Riverpod 2.x** (`flutter_riverpod ^2.5.0`) | 编译期安全、对 async 友好 | ✅ Phase 2 |
| 数据类 | **freezed + json_serializable** (`freezed ^2.4.0`) | 替代手写样板 | ✅ |
| 网络 | **`dart:io` Socket** (裸 TCP) | PTP/IP 是二进制协议，无 HTTP | ✅ |
| 本地 KV | **`shared_preferences ^2.3.0`** | 替代 iOS `UserDefaults` | ✅ Phase 2 |
| 字体 | **`google_fonts ^6.2.1`** | Instrument Serif (标题衬线) + DM Mono (等宽标签) | ✅ Phase 2 |
| 日志 | **`logging ^1.2.0`** | 替代 `Infrastructure/AppLogger.swift` | ✅ Phase 2 |
| 相册 | `gal` 包 + 自写 platform channel | Android 13+ scoped storage 必须处理 | ✅ Phase 3 |
| 通知 | `flutter_local_notifications` + `flutter_background_service` | 跨端进度提示 | ✅ Phase 3 |
| Wi-Fi 监听 | `connectivity_plus` + `network_info_plus` | 断线检测 | ✅ Phase 3 |
| Lint | `flutter_lints ^6.0.0` + 11 条 Phase 2 加强规则 | `dart analyze` 0 issues | ✅ Phase 2 |

### 1.1 选 Flutter 而非 KMP 的决策依据

| 维度 | Flutter | KMP |
|---|---|---|
| 双端 UI 一份代码 | ✅ | ❌ |
| 上手 / 工具链 | ✅ | ❌ (Gradle + Xcode 桥接坑多) |
| iOS Live Activity 体验 | ⚠️ 降级 | ✅ 保留原生 |
| 后台下载稳健性 | ⚠️ 包封装 | ✅ 原生 API |
| 包大小 | ⚠️ +5-10MB | ✅ +2MB |

**结论**：用户希望"双端同步出包"权重高 + **iOS 体验可降级**（已有原 iOS 项目作为参考），选 Flutter。

## 2. 分层架构

```
┌─────────────────────────────────────────┐
│  UI 层 (Features + App)                  │  ← Flutter Widgets + Riverpod
├─────────────────────────────────────────┤
│  服务层 (Services)                       │  ← 偏好 / 持久化 / 通知 / 后台 / Wi-Fi
├─────────────────────────────────────────┤
│  平台桥接层 (Platform Channels)           │  ← 相册 / 前台服务 / 原生方法
├─────────────────────────────────────────┤
│  协议层 (Protocol)                       │  ← PTP/IP 编解码 + 会话
├─────────────────────────────────────────┤
│  Domain 层 (Domain)                      │  ← freezed data classes, 无 IO
└─────────────────────────────────────────┘
```

**依赖方向**：上层可依赖下层，**下层绝不依赖上层**。Domain 层无任何外部依赖 (除 `dart:core` + `freezed_annotation`)。

**`platform/` 与 `services/` 边界**：`platform/` 只放 MethodChannel 接口定义 + 各端实现（薄壳，原生胶水层）；`services/` 调用 `platform/` 处理业务逻辑（厚壳，业务规则）。两者是**调用关系**，不是平级关系。例：`services/photo_library_export_service.dart` 调用 `platform/photo_library_channel_android.dart` 实现相册写入。

> 字段数量严格对齐原 iOS Swift。Phase 0（2026-07-22）验证通过；Phase 1 + Phase 2 未修改 Domain 层。

> 字段数量严格对齐原 iOS Swift。Phase 0（2026-07-22）验证通过。

## 5. 协议层设计要点（Phase 1 已完成，47 测试全绿）

### 5.1 PTP/IP 概述

- **CIPA PTP/IP** 标准协议，TCP 端口 **15740**
- 命令 / 响应 / 数据流三种 packet
- 命令 packet：12 字节 header (length + type + code + transactionId + params)
- 数据流：`GetObject` 返回大数据块 (单张照片 10-50 MB)

### 5.2 模块边界

| 模块 | 职责 | 不做 |
|---|---|---|
| `primitives/` | 单 packet 编解码 | socket IO |
| `transport/` | 长连接 + 心跳 + 重连 | 业务逻辑 |
| `session/ptpip_session.dart`（lifecycle） | OpenSession / CloseSession | 数据传输 |
| `session/ptpip_session.dart`（asset_traversal） | GetObjectHandles / GetObjectInfo | 下载 |
| `session/ptpip_session.dart`（transfers） | GetObject (单张大文件传输) | 列表浏览 |
| `camera_transport.dart` | 品牌抽象接口 | 具体实现 |
| `experimental_nikon_transport.dart` | Nikon opcode 包装 + _classifyObjectFormat + _mapError | 通用 PTP/IP |

### 5.3 异步模型

- `PtpipSocket` 是 abstract class，定义接口；`IoPtpipSocket` / `FakePtpipSocket` 是两种实现
  - **iOS / Android**：`dart:io.Socket`（BSD socket 跨端）
  - **测试**：fake 实现 (`FakePtpipSocket`，内存 buffer + scriptedResponses + scriptedErrors)
- `PtpipConnection` 持有 socket，处理异步收发 (Stream + Completer)
- `PtpipSession` 提供 `Future<List<PhotoAsset>> listAssets()` 等**协程风格 API**
- `ExperimentalNikonTransport` 把 `PtpipSession` 包成品牌适配层（错误映射 + 格式分类 + capability 推断）

**线程模型**：所有 PTP/IP 操作运行在 Dart isolate 的 main event loop；socket IO 用异步 listener，**不阻塞 UI**。

## 6. 状态管理 (Riverpod) 设计（Phase 3 落地）

### 6.1 Provider 拓扑（15+ Provider 全链路打通）

**Phase 3 核心新增**：8 个 service Provider + `cameraTransportProvider` + `cameraSessionProvider` + `downloadManagerNotifierProvider`。

| Provider | 类型 | 依赖 | 职责 |
|---|---|---|---|
| `sharedPreferencesProvider` | `Provider<SharedPreferences>` | — | SharedPreferences 单例 |
| `preferencesStoreProvider` | `Provider<AppPreferencesStore>` | `sharedPreferencesProvider` | JSON round-trip + schema 兼容 |
| `preferencesProvider` | `NotifierProvider<PreferencesNotifier, CameraConnectionConfig>` | `preferencesStoreProvider` | 设置读写 |
| `transportFactoryProvider` | `Provider<CameraTransportFactory>` | — | 工厂 |
| `cameraTransportProvider` | `Provider<CameraTransport>` | `transportFactoryProvider` | active transport 单例 |
| `cameraSessionProvider` | `StateProvider<CameraSession?>` | — | 当前会话 |
| `connectionProvider` | `NotifierProvider<ConnectionNotifier, ConnectionState>` | `transportFactoryProvider` + `cameraTransportProvider` | 连接状态机 |
| `galleryProvider` | `AsyncNotifierProvider<GalleryNotifier, GalleryState>` | `cameraSessionProvider` + `cameraTransportProvider` | 真实缩略图列表 |
| `downloadManagerNotifierProvider` | `NotifierProvider<DownloadManagerNotifier, DownloadQueueState>` | `cameraTransportProvider` + `cameraSessionProvider` + `downloadStoreProvider` + `preferencesProvider` | 队列全量管理 |
| `appShellProvider` | `NotifierProvider<AppShellNotifier, AppShellState>` | 监听 `connectionProvider` / `galleryProvider` | 全局 alerts / 日志 / overlay |
| `downloadStoreProvider` | `Provider<DownloadStore>` | `sharedPreferencesProvider` | 队列持久化 |
| `assetThumbnailServiceProvider` | `Provider<AssetThumbnailService>` | `cameraTransportProvider` + `cameraSessionProvider` | 缩略图 cache |
| `logFileStoreProvider` | `Provider<LogFileStore>` | — | 日志文件 |
| `notificationServiceProvider` | `Provider<DownloadNotificationService>` | — | 进度通知 |
| `backgroundRunnerProvider` | `Provider<BackgroundRunner>` | — | 后台执行 |
| `photoLibraryChannelProvider` | `Provider<PhotoLibraryChannel>` | — | 相册导出 |

### 6.2 Notifier 划分（6 个）

| Notifier | 类型 | 来源 |
|---|---|---|
| `ConnectionNotifier` | `Notifier<ConnectionState>`（9 字段 freezed） | 原 `ConnectionViewModel` |
| `GalleryNotifier` | `AsyncNotifier<GalleryState>`（freezed 包 selectedAssetIDs + isLoading） | 原 `GalleryViewModel` |
| `DownloadManagerNotifier` | `Notifier<DownloadQueueState>` + 13 公开方法 | 原 `DownloadManagerViewModel` |
| `PreferencesNotifier` | `Notifier<CameraConnectionConfig>` | 原 `SettingsViewModel` |
| `AppShellNotifier` | `Notifier<AppShellState>` | 原 `AppShellViewModel` |
| `GalleryConnectionNotifier` (内部) | 桥接 cameraSession → gallery 自动加载 | Phase 3 新增 |

### 6.3 关键设计原则 (Phase 3 更新)

- **协议层不持有 Riverpod**：`PtpipSession` 是 plain Dart 类
- **`ref.listen`** 跨 Provider 副作用
- **onProgress 回调链**：`ExperimentalNikonTransport.downloadAsset(onProgress:)` → `DownloadManagerNotifier._handleProgressUpdate()` → `DownloadNotificationService.update()`，不通过 Riverpod 广播进度（避免 UI 不必要 rebuild）
- **Provider 替代 Coordinator**：iOS `CameraSessionCoordinator` 职责由 Riverpod Provider 拓扑 + `WifiWatcher` 替代
- **测试用 `ProviderContainer` + `overrideWith`** 注入 fake transport
- **`unawaited(_save())`** fire-and-forget persistence
